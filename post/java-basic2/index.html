7. 数据类型

基本数据类型

1. 数值型 (默认int)

  • 声明long类型的数值时，需要注意赋值最后需要加上 “l” 或 “L”。一般写 “L” 清晰易于分辨。
名称 字节 位 范围
byte 1 8 -128 ~ 127 (-2^7 ~ 2^7-1)
short 2 16 -32768 ~ 32767 (-2^15 ~ 2^15-1)
int 4 32 -2^31 ~ 2^31-1
long 8 64 -2^63 ~ 2^63-1

2. 浮点型

  • 用于表示溢出和出错的情况的三个特殊浮点数值：
    • 正无穷
    • 负无穷
    • NaN（不是一个数字）
      • 正整数除以 0：正无穷
      • 0/0 或者负数的平方根结果：NaN
  • 声明float类型时，需要在赋值的值后面加上 “f” 或 “F”。
名称 字节 位 范围
float 4 32 有效数字6~7 -3.4E+38 ~ 3.4E+38
double 8 64 有效数字15~16 -1.7E-308 ~ 1.7E+308

3. 字符型

名称 字节 位 范围
char 2 16 0 ~ 65535

4. 布尔型(未定义时，默认false)

基本数据类型之间的转换

自动类型转换：小范围 → 大范围

  • byte → short → int → long → float → double
  • char → int → long → float → double
    • byte/short很容易自动转换为int类型，自我增值不会改变数据类型。

强制类型转换：

  • 需要加强制转换符：（希望修改的数据类型）当前数据类型

    short s = 10;
int num = (int)s;

double d = 1.0303
int num1 = (int)d;

  • 强制转换大范围 → 小范围，若最终结果精度大于小范围，会丢失精度。

大数

如果基本的整数和浮点数精度不够满足要求，可以使用 java.math 包中的两个类： BigInteger 和 BigDecimal。这两个类可以处理包含任意长度数字序列的数值。BigInteger 可以实现任意精度的整数运算，BigDecimal 类实现任意精度的浮点数运算。

使用静态的 valueOf() 方法可以将普通的数值转换为大数。

BigInteger a = BigInteger.valueOf(100);

对于更大的数字，可以使用一个带字符串参数的构造器。

BigInteger reallyBig = new BigInteger("2323232323232323232232323232323232323232323232322322323");

另外还有一些常量：
BigInteger.ZERO, BigInteger.ONE,BigInteger.TEN, Java 9 之后新增BigInteger.TWO。
不过需要注意的是，不能使用+，*算术符号处理大数，需要使用大数中的 add() 和 multiply() 方法。

BigInteger c = a.add(b);

与 C++不同， Java 无法重载+和*，只能使用大数自带的方法计算。

a = a * (n-i-1)/i;
a = a.multiply(BigInteger.valueOf(n - i -1).divide(BigInteger.valueOf(i)));

知识点补充

BigInteger

  1. BigInteger add(BigInteger other): 返回这个大整数与 other 的和。
  2. BigInteger subtract(BigInteger other): 返回这个大整数与 other 的差。
  3. BigInteger multiply(BigInteger other): 返回这个大整数与 other 的乘积。
  4. BigInteger divide(BigInteger other): 返回这个大整数与 other 的商。
  5. BigInteger mod(BigDInteger other): 返回这个大整数与 other 的余数。
  6. BigIntegr sqrt(): 得到这个大整数的平方根。
  7. int compareTo(BigInteger other): 如果大整数等于另一个大整数 other 返回 0；小于返回负数， 大于返回正数。
  8. static BigInteger valueOf(long x): 返回值等于 x 的一个大整数。

BigDecimal

  1. BigDecimal add(BigDecimal other): 返回这个大实数与 other 的和。
  2. BigDecimal subtract(BigDecimal other): 返回这个大实数与 other 的差。
  3. BigDecimal multiply(BigDecimal other): 返回这个大实数与 other 的乘积。
  4. BigDecimal divide(BigDecimal other): 返回这个大实数与 other 的商。
  5. BigDecimal divide(BigDecimal other, RoundingMode mode): 返回这个大实数与 other 的四舍五入的商。
    • 如果商是个无限循环小数，第一个 divide 方法会抛出一个异常。要得到四舍五入的结果，需要使用第二个除法方法。
    • RoundingMode.HALF_UP是我们以前数学学习里学到的四舍五入的方式。他适合用于常规的计算。
    • 作业：查询有关舍入的其他方式。
  6. int compareTo(BigDecimal other): 如果这个大实数与 other 相等， 返回 0；如果这个大实数小于 other， 返回复数；反之，大于返回正数。（是否为-1， 1）
  7. static BigDecimal valueOf(long x): 返回值等于 x 的一个大实数。
  8. static BigDecimal valueOf(long x, int scale): 返回值等于 x / 10 ^ scale 的一个大实数。

引用数据类型

1. 字符串 String

  • 从概念上来说，Java 字符串就是 Unicode 字符串序列。
  • Java 没有内置的字符串类型，而是在标准 Java 类库中提供了一个预定义类，很自然地叫做 String。
  • 每一个双引号里面的字符串都是 String 类的一个实例。
String str = "Java\u2122";
//Java™

子串 substring

  • String 类的 substring 方法可以从一个较大的字符串中提取出一个子串。
String str = "Hello";
// 创建一个由字符串”Hel“组成的字符串。到位置 3 为止，不包含 位置 3。
String sb_str = str.substring(0, 3);

字符串拼接

  • Java 允许用 + 来连接两个字符串或字符串 + 其他类型的数据（并将他们合并成一个字符串）
  • 把多个字符串放在一起，用一个界定符分隔，可以使用静态 join 方法：
String all = String.join("/", "S", "M", "L", "XL");

repeat()方法

String repeatStr = "Java".repeat(3);
//"JavaJavaJava"

如何修改原字符串，从而得到一个你想要的字符串

  • Java 中没有提供修改字符串中某个字符的方法。
  • 可以通过新创建一个希望留存的子字符串，再通过拼接的方式创建我们所需要的新字符串。
  • 由于不可修改 Java 字符串中某个单独的字符，所以 Java 文档中将 String 类对象称为不可变的（immutable）。

空串与 Null 串

  • 空串是长度为 0 的字符串：“”
  • 判断空串：
if(字符串名.length() == 0)
if(字符串名.equals(""))
// 两种方式皆可。
  • Null 串是无字符串值但是有这个名的字符串，仅声明并没有赋值。
  • 判断Null串：
if(字符串名 == null)
  • 检查既不是空串也不是 null
if(字符串名 ！= null && str.length() != 0)

字符串长度与得到特定位置的字符

int getCharLength = 字符串名.length();

//n -> 0 ~ 字符串名.length() - 1
//如果 n >= 字符串名.length()，就会报错：java.lang.StringIndexOutOfBoundsException
char getChar = 字符串名.charAt(n);

构建长字符串

  • 有时候需要进行构建字符串，由于无法直接加入或修改字符串，每次都需要新建一个字符串来存储原旧字符串 + 新的值/字符串，拼接效率非常的低。
  • 每次拼接都需要新建 String 对象，会耗费时间和空间。
  • 使用 StringBuilder 类就可以避免问题的发生。

如果需要一些小段的字符串来组成一个大的字符串：

  1. 构建一个空的字符串构造器
  2. 每次需要添加一部分内容时，就调用 append 方法。
  3. 在字符串构建完成时就调用 toString 方法，将可以得到一个 String 对象，其中包含构造器中的字符序列。
  StringBuilder str_builder = new StringBuilder();

  str_builder.append(字符名); 
  str_builder.append(字符串名);

  String completedString = str_builder.toString();

使用基本的控制台来实现输入

将输出打印到“标准输入流”（即控制台窗口）是一件非常容易的事情，只要调用System.out.println()即可。然而，读取“标准输入流” System.in就不没有那么简单了。想要通过控制台输入，首先需要构造一个与“标准输入流” System.in 相关联的 Scanner 对象

// 当使用的类不在 java.lang 包中时，需要导入其他依赖的包。
// Scanner 类在 java.util.* 包下
import java.util.Scanner;
//import java.util.*; 也是可以的，不过导入的是util下的所有类。
Scanner in = new Scanner(System.in);

现在就可以使用 Scanner 类下的各种方法来进行读取输入了。

System.out.println("你的名字是： ");
String name = in.nextLine();

在这里，使用nextLine() 方法是因为再输入行中有可能包含空格。想要读取一个单词（以空白符做为分隔符），可以调用

String first_name = in.next();

想要读取整数就调用nextInt()方法。

int age = in.nextInt();

作业：询问使用者姓名，年龄（可以自增喜欢的）。

使用Console类来实现读取密码

因为输入时可见的，所以 Scanner 类不适合从控制台中读取密码。 Java 6 特别引入了 Console 类来实现这个目的。 若想要读取一个密码，可以使用下列代码：

Console cons = System.console;
String username = cons.readLine("Username: ");
char[] password = cons.readPassword("Password: ");

为了安全起见，返回的密码必须存储于一个字符数组中， 而不是字符串中。在对密码处理完成后， 应该马上使用一个填充值覆盖数组元素。 采用 Console 对象处理输入不如采用 Scanner 方便。 必须每次读取一行输入， 而没有能够读取单个单词或数值的方法。

作业：

  1. 查看 java.util.Scanner 类下的各种方法，了解他们的构造与使用。
  2. 了解 java.lang.Console 类中的console()
    • 如果可以进行交互，就返回一个 Console 对象通过控制台窗口与用户进行交互， 否则返回null。 对于renhe一个在控制台窗口启动的程序， 都可以使用 Console 对象。否则，是否可用取决于所使用的系统。
  3. java.io.Console
String char[] readPassword(String prompt, Object... args)
static String readLine(String prompt, Object... args)

显示字符串prompt（提示符）并读取用户输入， 直到输入行结束。 args 参数可以用来提供格式参数。

格式化输出

可以使用 System.out.println(x); 将数值 x 输出到控制台。 这条命令将以 x 的类型所允许的最大非0 数个数打印输出 x。

double dollar = 10000.0 /3;
//3333.333333直到double的有效数字（15~16 位）
System.out.println(dollar);

这样会引起争议，毕竟美元最小单位是美分，并不能这样打印。所以在 Java 5 引入 C 函数中的 printf() 方法。

System.out.printf("%8.2f", x);

这里注意，“ ”内的内容并不会直接被打印出来。
他会以一个字宽（field width） 长度打印 x：

  • %: 每个以 % 开始的格式说明符都用相应的参数替换。 格式说明符尾部的转换符指示要格式化的参数的类型。
  • 8.2: 8个字符，精度为小数点后2字符。打印一个前导的空格和7个字符
    • 空格3333.33 [一个空格，3333，小数点，33]
  • f: 定点浮点数

String msg = String.format("Hellio, %s. Next year, you will be %d, name, age");

简单的文件输入与输出
  • 想要读取文件，需要构造一个 Scanner 对象。
Scanner in = new Scanner(Path.of("myfile.txt"), StandardCharsets.UTF_8);//指定了UTF8的字符编码，否则读取中文容易乱码

  • 现在就可以利用前面介绍的任何一个 Scanner 方法对文件进行读取。

  • 想要写入文件，就需要构造一个 PrinteWriter 对象。 构造器 （constructor） 中， 需要提供文件名和字符编码。

PrintWriter out = new PrintWriter("myfile.txt", StandardCharserts.UTF_8);

  • 如果文件不存在就创建该名字的文件。可以像输出到 System.out 一样使用 print, println 以及 printf 命令。

  • 可以构造一个带有字符串参数的 Scanner , 但这个 Scanner 会字符串解释为数据， 而不是文件名。

Scanner in = new Scanner("myfile.txt");

  • 这个Scanner会将参数看作包含10个字符（m y f 等）的数据。

  • 当你创建或读取一个文件时，文件的相对位置由 Java 虚拟机启动目录的位置决定。如果在命令行方式下执行 java MyProg 那么启动目录就是命令解释的当前目录。

  • 如果是ide 集成开发环境， 那么启动目录由ide 决定。 可以使用下面的调用找到这个目录的位置。或者打开ide找到你当前编译 .java文件目录下，相对位置就在这里了。

String dir = System.getProperty("user.dir");

  • 如果觉得文件定位太麻烦，也可以使用当前系统下的绝对路径。使用绝对路径时，不能换电脑！不能换电脑！不能换电脑！ 重要的事情说三遍。

  • 访问文件和 System.in 还有 System.out 一样容易， 但是如果一个不存在的文件构造一个 Scanner， 或者用一个无法创建的文件名狗仔一个 PrintWriter就会产生异常。 Java认为这些异常非常严重，所以我们需要加入处理异常的方式来告知我已经知道可能会产生异常。

public static void main(String []args) throws IOException{
  Scanner in new Scanner(Path.of("myfile.txt"), StandardCharsets.UTF_8);
}

3. 枚举 enum

  • 变量的取值有时候只在一个优先的集合内。
    • 举🌰：
      • 服装大小：小，中，大，超大
      • 可以设计成 1，2，3，4 或者 S，M，L，XL
      • 但是这种设计很容易出错，可能你保存了一个错误值：0 或 m
    • 针对这种情况，可以自定义枚举类型。枚举类型包括有限个命名的值。
enum Size{SMALL, MDEIUM, LARGE, EXTRA_LARGE};
// 声明这种类型的变量
Size s = Size.MEDIUM;
  • Size 类型的变量只能存储这个类声明中给定的某个枚举值，或者特殊值 null。
  • null表示这个变量没有设置任何值。
